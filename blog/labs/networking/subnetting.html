Step-by-Step Process

01
Subnet Calculations

Learn the formula: hosts = 2^(32-prefix) - 2. Practice with common prefixes.

# Common prefix sizes:
/24 = 256 addr, 254 usable
/25 = 128 addr, 126 usable
/26 = 64  addr, 62 usable
/27 = 32  addr, 30 usable
/28 = 16  addr, 14 usable
/30 = 4   addr, 2 usable (point-to-point links)
/32 = host route

ipcalc 192.168.10.0/26    # detailed breakdown
02
VLSM — Variable Length Subnet Masking

Allocate subnet sizes based on actual needs to avoid address waste.

# Given: 10.0.0.0/24, design for:
# HQ: 50 hosts  → need /26 (62 usable)
# Branch: 25 hosts → need /27 (30 usable)
# Server: 10 hosts → need /28 (14 usable)
# WAN link: 2 hosts → need /30

# Allocate from largest to smallest:
# 10.0.0.0/26  ← HQ (50 hosts)
# 10.0.0.64/27 ← Branch
# 10.0.0.96/28 ← Server
# 10.0.0.112/30 ← WAN
03
Python for Subnet Math

Use Python's ipaddress module for programmatic subnet calculation.

python3
import ipaddress

n = ipaddress.ip_network('192.168.1.0/24')
print(f'Network: {n.network_address}')
print(f'Broadcast: {n.broadcast_address}')
print(f'Hosts: {n.num_addresses - 2}')

# Subnets of size /26
for s in n.subnets(new_prefix=26):
    print(s, '-', list(s.hosts())[0], 'to', list(s.hosts())[-1])

Challenges & Solutions

  • Off-by-one: subtract 2 (network + broadcast) from total addresses for usable
  • Overlapping subnets cause routing ambiguity — check with ipcalc

Key Takeaways

  • Always allocate from largest subnet to smallest to avoid gaps
  • A /30 wastes 2 addresses — /31 works for point-to-point per RFC 3021
