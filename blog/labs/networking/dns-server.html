DNS Server Configuration

Intermediate Networking
BIND9namedzonesforward zonereverse zonedig

Objective

Install BIND9, create forward and reverse zone files, configure resolution, and troubleshoot with dig and nslookup.

Tools & Technologies

  • BIND9
  • named
  • zones
  • forward zone
  • reverse zone
  • dig

Key Commands

apt install bind9
named-checkconf
named-checkzone lab.local /etc/bind/db.lab.local
dig @localhost lab.local
dig -x 192.168.1.10

Architecture Overview

Step-by-Step Process

01
Install and Configure BIND9

Install BIND9 and configure the named.conf options.

sudo apt install -y bind9 bind9utils dnsutils

# /etc/bind/named.conf.options
options {
  directory "/var/cache/bind";
  recursion yes;
  allow-query { localhost; 192.168.1.0/24; };
  forwarders { 1.1.1.1; 8.8.8.8; };
  dnssec-validation auto;
};
02
Create Forward Zone

A forward zone maps hostnames to IP addresses.

# /etc/bind/named.conf.local
zone "lab.local" {
  type master;
  file "/etc/bind/db.lab.local";
};

# /etc/bind/db.lab.local
$TTL 86400
@ IN SOA ns1.lab.local. admin.lab.local. (
  2024010101 ; Serial
  3600       ; Refresh
  900        ; Retry
  604800     ; Expire
  86400 )    ; Negative TTL

@ IN NS ns1.lab.local.
ns1   IN A 192.168.1.1
www   IN A 192.168.1.50
mail  IN A 192.168.1.51
mail  IN MX 10 mail.lab.local.
03
Create Reverse Zone

Reverse zones map IPs to hostnames (PTR records).

# /etc/bind/named.conf.local
zone "1.168.192.in-addr.arpa" {
  type master;
  file "/etc/bind/db.192.168.1";
};

# /etc/bind/db.192.168.1
$TTL 86400
@ IN SOA ns1.lab.local. admin.lab.local. (
  2024010101 3600 900 604800 86400)
@ IN NS ns1.lab.local.
1  IN PTR ns1.lab.local.
50 IN PTR www.lab.local.
51 IN PTR mail.lab.local.
04
Test & Verify

Use named-check tools and dig to verify the configuration.

# Validate config and zone files
sudo named-checkconf
sudo named-checkzone lab.local /etc/bind/db.lab.local
sudo named-checkzone 1.168.192.in-addr.arpa /etc/bind/db.192.168.1

sudo systemctl restart bind9

# Test queries
dig @localhost lab.local NS
dig @localhost www.lab.local A
dig @localhost -x 192.168.1.50
nslookup www.lab.local 127.0.0.1

Challenges & Solutions

  • Serial number must be incremented on every zone change or secondaries won't update
  • named-checkzone reports errors that would silently prevent zone loading

Key Takeaways

  • Start named with -g in foreground to see startup errors: named -g -u bind
  • Allow-recursion should not be open to 0.0.0.0/0 — prevents DNS amplification attacks
