OpenFlow Protocol

Advanced Networking
OpenFlowOVSflow tablesmatch-actionovs-ofctl

Objective

Understand OpenFlow flow tables, install flow entries manually using ovs-ofctl, and observe how packets are matched and forwarded.

Tools & Technologies

  • OpenFlow
  • OVS
  • flow tables
  • match-action
  • ovs-ofctl

Key Commands

ovs-vsctl show
ovs-ofctl dump-flows br0
ovs-ofctl add-flow br0 'priority=100,ip,nw_dst=10.0.0.2,actions=output:2'
ovs-ofctl del-flows br0

Architecture Overview

Step-by-Step Process

01
Open vSwitch Basics

OVS is a production-quality software switch that supports OpenFlow.

# Install
sudo apt install openvswitch-switch

# Create a bridge
ovs-vsctl add-br br0
ovs-vsctl add-port br0 eth1
ovs-vsctl show

# Connect to controller
ovs-vsctl set-controller br0 tcp:127.0.0.1:6633
02
Inspect and Add Flow Entries

Flow entries have match fields and actions.

# View flow table
ovs-ofctl dump-flows br0

# Add flow: forward IP to 10.0.0.2 out port 2
ovs-ofctl add-flow br0 \
  'priority=100,ip,nw_dst=10.0.0.2,actions=output:2'

# Add flow: flood ARP
ovs-ofctl add-flow br0 \
  'priority=200,arp,actions=flood'

# Drop traffic from a source
ovs-ofctl add-flow br0 \
  'priority=500,ip,nw_src=10.0.0.99,actions=drop'
03
Timeouts and Statistics

Flow entries can have idle and hard timeouts. Monitor match counts.

# Add flow with timeout
ovs-ofctl add-flow br0 \
  'priority=100,ip,idle_timeout=30,hard_timeout=120,actions=output:2'

# View statistics (match counts)
ovs-ofctl dump-flows br0

# Port statistics
ovs-ofctl dump-ports br0

Challenges & Solutions

  • Table-miss flows must be configured — without one, unmatched packets are dropped
  • Flow priority determines match precedence — higher number = higher priority

Key Takeaways

  • Each flow entry consumes TCAM space in hardware switches — keep tables lean
  • ovs-ofctl dump-ports shows per-port packet/byte counters
