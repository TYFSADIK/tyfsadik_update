Network Function Virtualization

Advanced Networking

Objective

Understand NFV concepts, implement network service chaining with Linux network namespaces, and deploy a software load balancer as a VNF.

Step-by-Step Process

01
Network Namespaces as VNFs

Linux network namespaces are isolated network stacks — the foundation of containers and VNFs.

# Create namespace
ip netns add firewall
ip netns add backend

# Create veth pair (virtual Ethernet)
ip link add veth-fw type veth peer name veth-fw-ns
ip link set veth-fw-ns netns firewall

# Configure inside namespace
ip netns exec firewall ip addr add 10.0.1.1/24 dev veth-fw-ns
ip netns exec firewall ip link set veth-fw-ns up
ip netns exec firewall ip link set lo up
02
Deploy HAProxy as Load Balancer VNF

Run HAProxy inside a namespace as a virtual load balancer function.

sudo apt install haproxy

# /etc/haproxy/haproxy.cfg
frontend web
  bind *:80
  default_backend servers

backend servers
  balance roundrobin
  server s1 192.168.1.10:80 check
  server s2 192.168.1.11:80 check

sudo haproxy -f /etc/haproxy/haproxy.cfg -c  # check config
sudo systemctl start haproxy
03
Service Function Chaining

Chain network functions so traffic flows through each VNF in sequence.

# Chain: Client → Firewall NS → LB NS → Backend
# Create namespaces and veth pairs
ip netns add lb
ip link add veth-lb1 type veth peer name veth-lb1-ns
ip link set veth-lb1-ns netns lb

# Enable routing between namespaces
ip netns exec firewall sysctl -w net.ipv4.ip_forward=1

# Add iptables rules in firewall namespace
ip netns exec firewall iptables -A FORWARD -j ACCEPT

Challenges & Solutions

  • veth pairs are always created in pairs — one end per namespace
  • Forgetting ip link set lo up inside namespace causes localhost failures

Key Takeaways

  • Network namespaces underpin Docker networking — each container gets its own namespace
  • NFV reduces hardware costs and enables rapid service deployment
