Wireshark Packet Analysis

Intermediate Networking
Wiresharktcpdumptsharkpcapdisplay filters

Objective

Capture and analyze network traffic using Wireshark and tcpdump, apply display filters, and decode protocol behaviour.

Tools & Technologies

  • Wireshark
  • tcpdump
  • tshark
  • pcap
  • display filters

Key Commands

tcpdump -i eth0 -w capture.pcap
tshark -r capture.pcap -Y 'http'
wireshark capture.pcap
tshark -i eth0 -f 'port 80' -c 100

Architecture Overview

Step-by-Step Process

01
Capture with tcpdump

Use tcpdump to capture to a file, then open in Wireshark.

# Capture 1000 packets on eth0
tcpdump -i eth0 -c 1000 -w lab.pcap

# BPF capture filters
tcpdump -i eth0 'port 80'           # HTTP only
tcpdump -i eth0 'host 10.0.0.5'     # specific host
tcpdump -i eth0 'tcp and port 443'  # HTTPS
tcpdump -i eth0 -nn               # no DNS/port resolution
02
Wireshark Display Filters

Display filters let you narrow down packets after capture.

# Common display filters:
ip.addr == 192.168.1.5
tcp.port == 80
http
dns
tcp.flags.syn == 1           # SYN packets only
http.response.code == 404    # HTTP 404s
tls.handshake.type == 1      # TLS ClientHello

# Combine with &&, ||, !
ip.addr == 10.0.0.5 && tcp.port == 443
03
Analyse Common Protocols

Use Wireshark's protocol dissectors and Follow Stream feature.

# HTTP: Right-click → Follow → TCP Stream
# Shows full request+response as text

# DNS: filter dns
# See query/response pairs, latency

# TCP handshake: filter tcp.flags.syn==1
# Statistics → Conversations
# Statistics → Protocol Hierarchy

Challenges & Solutions

  • Promiscuous mode may not work inside VMs or on wireless adapters
  • TLS 1.3 prevents decryption without pre-master secret key log

Key Takeaways

  • Use capture filters (BPF) to reduce file size — display filters cannot reduce capture overhead
  • Statistics → Expert Information shows retransmissions, resets, and errors automatically
