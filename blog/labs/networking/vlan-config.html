VLAN Configuration

Intermediate Networking
Cisco IOS802.1Qtrunkaccess portsip link

Objective

Create and configure VLANs on a managed switch, configure trunk and access ports, and create VLAN subinterfaces on Linux.

Tools & Technologies

  • Cisco IOS
  • 802.1Q
  • trunk
  • access ports
  • ip link

Key Commands

vlan 10
switchport access vlan 10
switchport mode trunk
ip link add link eth0 name eth0.10 type vlan id 10
ip addr add 10.10.0.1/24 dev eth0.10

Architecture Overview

Step-by-Step Process

01
Create VLANs on a Switch (Cisco IOS)

Create VLANs in the VLAN database and assign access ports.

! On Cisco switch
conf t
vlan 10
 name HR
vlan 20
 name Engineering
vlan 30
 name Printers
exit

! Assign access port
interface Fa0/1
 switchport mode access
 switchport access vlan 10
02
Configure Trunk Ports

Trunk ports carry multiple VLANs between switches and to routers.

interface Gi0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30

! Verify
show interfaces trunk
show vlan brief
03
Linux VLAN Subinterfaces

Create 802.1Q VLAN subinterfaces on a Linux host connected to a trunk port.

# Load 8021q module
sudo modprobe 8021q

# Create VLAN 10 subinterface
ip link add link eth0 name eth0.10 type vlan id 10
ip link set eth0.10 up
ip addr add 10.10.0.1/24 dev eth0.10

# Create VLAN 20
ip link add link eth0 name eth0.20 type vlan id 20
ip link set eth0.20 up
ip addr add 10.20.0.1/24 dev eth0.20
04
Verify VLAN Isolation

Confirm hosts on different VLANs cannot communicate without routing.

# From VLAN 10 host
ping 10.10.0.2   # same VLAN — should work
ping 10.20.0.2   # different VLAN — should FAIL

# Capture traffic to verify tagging
tcpdump -i eth0 -e 'vlan'

# Cisco verification
show mac address-table
show vlan brief

Challenges & Solutions

  • Native VLAN mismatch between trunk ports causes untagged traffic issues
  • Forgetting to trunk-allow a VLAN blocks it across the link

Key Takeaways

  • VLANs provide Layer 2 isolation without physical cabling changes
  • Access ports strip VLAN tags — hosts see untagged frames
