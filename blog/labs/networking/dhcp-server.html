DHCP Server Configuration

Intermediate Networking
isc-dhcp-serverdhcpd.confDORA processstatic leases

Objective

Install and configure a DHCP server, define scopes, configure static leases, and understand the DORA discovery process.

Tools & Technologies

  • isc-dhcp-server
  • dhcpd.conf
  • DORA process
  • static leases

Key Commands

apt install isc-dhcp-server
systemctl start isc-dhcp-server
dhcp-lease-list
journalctl -u isc-dhcp-server

Architecture Overview

sequenceDiagram participant C as Client participant S as DHCP Server C->>+S: DISCOVER (broadcast) Note over C: No IP yet, src: 0.0.0.0 S-->>-C: OFFER (unicast/broadcast) Note over S: Offers 192.168.1.50 C->>+S: REQUEST (broadcast) Note over C: Accepts the offer S-->>-C: ACK (unicast) Note over S: Confirms lease C->>C: Configure interface Note over C: Now has 192.168.1.50/24

Step-by-Step Process

01
Install & Configure

Configure the DHCP server with subnet, range, and options.

sudo apt install -y isc-dhcp-server

# /etc/dhcp/dhcpd.conf
default-lease-time 86400;
max-lease-time 604800;

subnet 192.168.1.0 netmask 255.255.255.0 {
  range 192.168.1.100 192.168.1.200;
  option routers 192.168.1.1;
  option domain-name-servers 1.1.1.1, 8.8.8.8;
  option domain-name "lab.local";
}
02
Static Leases (Reservations)

Assign a specific IP to a device based on its MAC address.

# In dhcpd.conf
host printer {
  hardware ethernet 00:1A:2B:3C:4D:5E;
  fixed-address 192.168.1.20;
}

host server01 {
  hardware ethernet AA:BB:CC:DD:EE:FF;
  fixed-address 192.168.1.10;
}
03
Test & Monitor

Verify the server starts and leases are being issued.

# Set listening interface
# /etc/default/isc-dhcp-server:
# INTERFACESv4="eth0"

sudo systemctl restart isc-dhcp-server
sudo systemctl status isc-dhcp-server

# View active leases
cat /var/lib/dhcpd/dhcpd.leases
dhcp-lease-list

# Watch logs
journalctl -u isc-dhcp-server -f

Challenges & Solutions

  • Two DHCP servers on same subnet cause conflicts — ensure only one is active
  • Wrong interface in /etc/default/isc-dhcp-server prevents server from binding

Key Takeaways

  • Set lease time based on network size — short leases for large dynamic networks
  • DHCP relay agent (ip helper-address on Cisco) allows serving multiple subnets from one server
