IP Routing Fundamentals

Intermediate Networking
ip routetraceroutenetstatstatic routes

Objective

Understand how routing decisions are made, add static routes, and analyze the routing table.

Tools & Technologies

  • ip route
  • traceroute
  • netstat
  • static routes

Key Commands

ip route show
ip route add 10.0.0.0/8 via 192.168.1.1
traceroute -n 8.8.8.8
ss -rn

Architecture Overview

Step-by-Step Process

01
Routing Table Basics

The routing table maps destination networks to next-hop gateways. Longest prefix match wins.

# View routing table
ip route show
netstat -rn

# Interpret output:
# default via 192.168.1.1 dev eth0  ← default route
# 192.168.1.0/24 dev eth0 proto kernel  ← connected network
# 10.0.0.0/8 via 192.168.1.1  ← static route
02
Add Static Routes

Add routes to reach networks that are not directly connected.

# Add static route
ip route add 10.0.0.0/8 via 192.168.1.1
# Add route via specific interface
ip route add 172.16.0.0/12 via 192.168.2.1 dev eth1
# Delete route
ip route del 10.0.0.0/8
# Persistent (add to netplan or nmcli config)
03
Trace the Path

Use traceroute/mtr to follow the path packets take.

traceroute -n 8.8.8.8     # numeric (no DNS)
mtr --report 8.8.8.8       # combines ping + traceroute
tracepath google.com       # no root needed

Challenges & Solutions

  • Most specific route (longest prefix) always wins over default route
  • Missing default route causes all non-local traffic to drop

Key Takeaways

  • traceroute * * * means ICMP is blocked at that hop — not necessarily a problem
  • ip route get 8.8.8.8 shows exactly which route would be used
