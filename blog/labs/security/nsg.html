Network Security Groups

Intermediate Security
Azure NSGAWS Security Groupsinbound rulesoutbound rulespriority

Objective

Create and manage cloud network security groups in Azure, understand rule priority, and test effective security rules.

Tools & Technologies

  • Azure NSG
  • AWS Security Groups
  • inbound rules
  • outbound rules
  • priority

Key Commands

az network nsg create
az network nsg rule create --priority 100
az network nsg show-effective-security-rules
aws ec2 authorize-security-group-ingress

Step-by-Step Process

01
Create NSG and Rules

Create a network security group with allow rules.

# Create NSG
az network nsg create \
  --resource-group MyRG \
  --name MyNSG

# Allow SSH from specific IP
az network nsg rule create \
  --resource-group MyRG \
  --nsg-name MyNSG \
  --name AllowSSH \
  --priority 100 \
  --protocol Tcp \
  --destination-port-ranges 22 \
  --source-address-prefixes 'MY.IP.ADDRESS/32' \
  --access Allow
02
Associate and Verify

Attach NSG to subnet or NIC and check effective rules.

# Associate to subnet
az network vnet subnet update \
  --resource-group MyRG \
  --vnet-name MyVNet \
  --name MySubnet \
  --network-security-group MyNSG

# Check effective rules on a NIC
az network nic show-effective-security-rules \
  --resource-group MyRG \
  --name MyNIC

Challenges & Solutions

  • NSG rules changes can take 1-2 minutes to propagate
  • Subnet NSG + NIC NSG: both must allow traffic for it to flow

Key Takeaways

  • Always restrict SSH/RDP to specific IPs — never 0.0.0.0/0 in production
  • Azure Network Watcher IP Flow Verify tests NSG rules without actual traffic
