Step-by-Step Process

01
Python Triage Script

Automate initial evidence collection.

#!/usr/bin/env python3
# ir_triage.py — collect volatile data
import subprocess, datetime, argparse, json

parser = argparse.ArgumentParser()
parser.add_argument('--host', required=True)
args = parser.parse_args()

timestamp = datetime.datetime.utcnow().isoformat()
collections = {
    'timestamp': timestamp,
    'target': args.host,
    'connections': subprocess.getoutput('ss -tunp'),
    'processes': subprocess.getoutput('ps auxf'),
    'users': subprocess.getoutput('last -n 20'),
}
print(json.dumps(collections, indent=2))
02
TheHive API Integration

Create incidents automatically from detection events.

import requests

hive_url = 'http://localhost:9000'
headers = {'Authorization': 'Bearer YOUR_TOKEN', 'Content-Type': 'application/json'}

alert = {
    'title': 'Failed SSH brute force',
    'description': '50+ failed logins from 1.2.3.4',
    'type': 'external',
    'source': 'fail2ban',
    'sourceRef': 'fail2ban-001',
    'severity': 2,
    'tags': ['ssh', 'brute-force']
}
resp = requests.post(f'{hive_url}/api/alert', json=alert, headers=headers)
print(resp.json())

Challenges & Solutions

  • Automated containment can cause false positive service outages — require human approval for destructive actions
  • API keys in scripts — use environment variables or vault, never hardcode

Key Takeaways

  • Start with enrichment automation before response automation — safer
  • Document automated actions in the incident ticket — audit trail is essential
