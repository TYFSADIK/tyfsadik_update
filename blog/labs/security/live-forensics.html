Live Box Forensics

Advanced Security
volatilityddlsofps auxfnetstatmemory acquisition

Objective

Collect volatile evidence from a running system: memory dump, network state, process list, and open files — before powering down.

Tools & Technologies

  • volatility
  • dd
  • lsof
  • ps auxf
  • netstat
  • memory acquisition

Key Commands

dd if=/proc/kcore of=/external/mem.dump
lsof -p PID
ps auxf
netstat -tunp
find / -newer /tmp -mtime -1 2>/dev/null

Architecture Overview

flowchart TD LIVE[Compromised System\nSTILL RUNNING] --> ORDER{Evidence Collection\nOrder of Volatility} ORDER --> CPU[CPU registers &\ncache - snapshot] CPU --> MEM[RAM / Memory\ndd or LiME] MEM --> NET[Network state\nss + netstat] NET --> PROC[Processes\nps + lsof] PROC --> DISK[Disk artifacts\nfiles, logs] DISK --> POWER[Power off\n+ image disk] style LIVE fill:#1a1a2e,stroke:#ff4444,color:#ff4444 style MEM fill:#1a1a2e,stroke:#00d4ff,color:#e0e0e0

Step-by-Step Process

01
Memory Acquisition

Capture RAM before it changes.

# Using LiME (requires kernel module)
git clone https://github.com/504ensicsLabs/LiME
cd LiME/src && make
sudo insmod lime-$(uname -r).ko 'path=/external/mem.lime format=lime'

# Partial via /proc/kcore (may not work on all systems)
sudo dd if=/proc/kcore of=/external/mem.dump bs=1M count=1024
02
Capture Network State

Document all current connections before they change.

# All network connections
ss -tunp > /external/ir_$(date +%s)_network.txt
# ARP table
arp -n >> /external/ir_$(date +%s)_arp.txt
# Routing table
ip route show >> /external/ir_$(date +%s)_routes.txt
# DNS cache
systemd-resolve --statistics
03
Process & File Analysis

Document running processes and open file handles.

# Full process tree
ps auxf > /external/ir_processes.txt

# Open files by suspicious process
lsof -p $(pgrep suspicious) > /external/ir_lsof.txt

# Files modified in last 24 hours
find / -newer $(date -d '24 hours ago' +/tmp/reffile) \
  -not -path '/proc/*' -not -path '/sys/*' 2>/dev/null

Challenges & Solutions

  • Writing forensic data to the compromised system modifies it — always write to external media
  • Some rootkits hide processes from ps — cross-reference with /proc directly

Key Takeaways

  • Collect in order of volatility: most volatile (memory, network) before less volatile (disk)
  • Document every action with timestamp — your notes may be used as evidence in court
