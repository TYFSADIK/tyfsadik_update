Dead Box Forensics

Advanced Security
Autopsyddsha256sumThe Sleuth Kitwrite blocker

Objective

Image a powered-off disk, verify integrity with hashing, and perform filesystem analysis using Autopsy and TSK tools.

Tools & Technologies

  • Autopsy
  • dd
  • sha256sum
  • The Sleuth Kit
  • write blocker

Key Commands

dd if=/dev/sda of=/evidence/disk.dd bs=4M conv=noerror,sync
sha256sum /evidence/disk.dd > /evidence/disk.dd.sha256
autopsy
fls -r -m / disk.dd

Architecture Overview

Step-by-Step Process

01
Create Forensic Image

Image the disk and record the hash immediately.

# Image with hash verification
sudo dcfldd if=/dev/sdb of=/evidence/disk.dd \
  bs=4M conv=noerror,sync \
  hashwindow=1G hashlog=/evidence/hash.log

# Or with dd + separate hash
sudo dd if=/dev/sdb of=/evidence/disk.dd bs=4M conv=noerror,sync
sha256sum /evidence/disk.dd > /evidence/disk.dd.sha256
02
Mount and Analyse

Mount the image read-only for safe analysis.

# Mount read-only
sudo mount -o ro,loop,noexec /evidence/disk.dd /mnt/evidence

# TSK: list files including deleted
fls -r -m / /evidence/disk.dd > /evidence/bodyfile.txt

# TSK: generate timeline
mactime -b /evidence/bodyfile.txt -d > /evidence/timeline.csv
03
Autopsy GUI Analysis

Use Autopsy for graphical file analysis.

sudo autopsy

Challenges & Solutions

  • Never analyse original media — always work from a verified image copy
  • Hash must be computed immediately after imaging — before any analysis

Key Takeaways

  • MD5 is no longer cryptographically secure — use SHA-256 for court-admissible evidence
  • Deleted files are recoverable until blocks are overwritten — image promptly
