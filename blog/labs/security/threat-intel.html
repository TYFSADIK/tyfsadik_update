Step-by-Step Process

01
MITRE ATT&CK Navigator

Map observed attacker techniques to the ATT&CK framework.

# Access: https://mitre-attack.github.io/attack-navigator/
# Key tactic categories:
# TA0001: Initial Access
# TA0002: Execution
# TA0003: Persistence
# TA0008: Lateral Movement
# TA0010: Exfiltration

# Example technique: T1078 Valid Accounts (privilege abuse)
# Sub-technique: T1078.003 Local Accounts
02
Query OTX for IOCs

Use AlienVault OTX free threat intelligence API.

import requests

API_KEY = 'your-otx-api-key'
IP = '1.2.3.4'

resp = requests.get(
    f'https://otx.alienvault.com/api/v1/indicators/IPv4/{IP}/general',
    headers={'X-OTX-API-KEY': API_KEY}
)
data = resp.json()
print(f"Reputation: {data.get('reputation')}")
print(f"Pulse count: {data['pulse_info']['count']}")
03
Integrate IOCs into Detection

Feed threat indicators into SIEM and firewall rules.

# Extract malicious IPs from OTX pulse
# Add to iptables block list
while IFS= read -r ip; do
  iptables -A INPUT -s "$ip" -j DROP
done < malicious_ips.txt

# Splunk: create lookup table from IOC feed
| inputlookup malicious_ips.csv
| rename IP as src_ip
| join src_ip [search index=network]

Challenges & Solutions

  • IOCs expire quickly — stale indicators generate false positives
  • Context matters: an IP used by a threat actor may also be shared infrastructure

Key Takeaways

  • TTPs are more valuable than IOCs — attackers change IPs and hashes, not techniques
  • STIX 2.1 is the standard for structured threat intelligence exchange
