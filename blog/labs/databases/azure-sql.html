Azure SQL Deployment

Intermediate Databases

Objective

Deploy an Azure SQL Database, configure firewall rules, and connect with management tools.

Tools & Technologies

  • Azure SQL
  • az sql
  • SSMS
  • sqlcmd
  • firewall rules

Key Commands

az sql server create --name myserver --resource-group rg1
az sql db create --name mydb --server myserver
az sql server firewall-rule create --name AllowMyIP
sqlcmd -S server.database.windows.net -U admin -P pass

Step-by-Step Process

01
Provision via Azure CLI

Create the logical server and database.

az sql server create \
  --name myserver-$(date +%s) \
  --resource-group MyRG \
  --location eastus \
  --admin-user sqladmin \
  --admin-password 'P@ssword123!'

az sql db create \
  --resource-group MyRG \
  --server myserver \
  --name mydb \
  --edition GeneralPurpose \
  --compute-model Serverless
02
Configure Access

Set firewall rules and test connectivity.

# Allow your IP
az sql server firewall-rule create \
  --resource-group MyRG \
  --server myserver \
  --name AllowMyIP \
  --start-ip-address $(curl -s ifconfig.me) \
  --end-ip-address $(curl -s ifconfig.me)

# Test with sqlcmd
sqlcmd -S myserver.database.windows.net -U sqladmin -P 'P@ssword123!' -d mydb -Q 'SELECT @@VERSION'

Challenges & Solutions

  • Firewall rule takes ~2 minutes to propagate
  • Password must meet complexity requirements or create fails with unhelpful error

Key Takeaways

  • Use Azure AD authentication in production — SQL auth passwords are less secure
  • Enable Threat Detection and Audit on production Azure SQL databases
