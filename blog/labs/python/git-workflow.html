Git Workflow for Python

Beginner Python
gitGitHub.gitignorebranchespre-commit

Objective

Set up a Python project with proper Git workflow, .gitignore, and understand branching.

Tools & Technologies

  • git
  • GitHub
  • .gitignore
  • branches
  • pre-commit

Key Commands

git init
git add .
git commit -m 'feat: initial commit'
echo '__pycache__/\n*.pyc\n.venv/' > .gitignore
git branch feature/new-feature

Architecture Overview

gitGraph commit id: "Initial commit" commit id: "Add requirements.txt" branch feature/auth commit id: "Add login function" commit id: "Add tests" checkout main merge feature/auth id: "Merge auth feature" commit id: "Release v1.0"

Step-by-Step Process

01
Initialize and First Commit

Set up the repo with proper Python .gitignore.

git init
git config user.name 'Your Name'
git config user.email 'you@example.com'

# Create .gitignore
cat > .gitignore << 'EOF'
.venv/
__pycache__/
*.pyc
*.pyo
.env
*.egg-info/
dist/
build/
.pytest_cache/
EOF

git add .gitignore README.md
git commit -m 'feat: initial project structure'
02
Feature Branch Workflow

Work on features in branches, merge via pull request.

# Create and switch to branch
git checkout -b feature/user-auth

# Work, then commit
git add auth.py
git commit -m 'feat: add JWT authentication'

# Push to remote
git push origin feature/user-auth

# Merge (after PR review)
git checkout main
git merge feature/user-auth
git push origin main

Challenges & Solutions

  • Committing without .gitignore accidentally includes .venv (100MB+)
  • Committing .env files exposes secrets — they should always be in .gitignore

Key Takeaways

  • Commit messages should explain WHY, not WHAT (the diff shows what)
  • Use conventional commits: feat:, fix:, docs:, refactor:
