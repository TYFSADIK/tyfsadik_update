Memory Usage Analyzer

Intermediate Python
/proc/meminfotracemallocpsutilargparsetabulate

Objective

Build a command-line memory analyzer that parses /proc/meminfo, profiles Python object allocations, and generates reports.

Tools & Technologies

  • /proc/meminfo
  • tracemalloc
  • psutil
  • argparse
  • tabulate

Step-by-Step Process

01
Parse /proc/meminfo

Read system memory statistics from the Linux virtual filesystem.

def parse_meminfo():
    mem = {}
    with open('/proc/meminfo') as f:
        for line in f:
            key, val = line.split(':', 1)
            mem[key.strip()] = int(val.split()[0])  # kB
    return mem

info = parse_meminfo()
total = info['MemTotal'] // 1024
free = info['MemFree'] // 1024
avail = info['MemAvailable'] // 1024
print(f'Total: {total} MB, Available: {avail} MB')
02
Profile Python Allocations

Use tracemalloc to find what's consuming memory.

import tracemalloc

tracemalloc.start()

# Code to profile
big_list = [x**2 for x in range(100000)]

snapshot = tracemalloc.take_snapshot()
top_stats = snapshot.statistics('lineno')

for stat in top_stats[:5]:
    print(stat)
03
Full CLI Tool

Combine both sources into a CLI with argparse.

#!/usr/bin/env python3
import argparse, psutil, tracemalloc

def main():
    parser = argparse.ArgumentParser(description='Memory usage analyzer')
    parser.add_argument('--top', type=int, default=5)
    parser.add_argument('--output', default='-', help='file or - for stdout')
    args = parser.parse_args()

    proc = psutil.Process()
    mem = proc.memory_info()
    print(f'RSS: {mem.rss//1024//1024} MB')
    print(f'VMS: {mem.vms//1024//1024} MB')

if __name__ == '__main__':
    main()

Challenges & Solutions

  • tracemalloc overhead is significant — don't leave it running in production
  • /proc/meminfo values are in kB — always convert for display

Key Takeaways

  • sys.getsizeof() only counts the object itself, not referenced objects
  • Use objgraph for a visual breakdown of object references
