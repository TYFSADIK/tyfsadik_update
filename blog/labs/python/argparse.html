Step-by-Step Process

01
Basic Argument Parsing

Define and parse command-line arguments.

import argparse

parser = argparse.ArgumentParser(
    description='Process log files',
    formatter_class=argparse.RawDescriptionHelpFormatter
)
# Positional (required)
parser.add_argument('filename', help='log file to process')
# Optional with default
parser.add_argument('-o', '--output', default='report.txt', help='output file')
parser.add_argument('-n', '--lines', type=int, default=100)
parser.add_argument('-v', '--verbose', action='store_true')

args = parser.parse_args()
print(args.filename, args.verbose)
02
Subcommands

Implement git-style subcommands.

parser = argparse.ArgumentParser()
subparsers = parser.add_subparsers(dest='command', required=True)

# 'check' subcommand
check = subparsers.add_parser('check', help='check system health')
check.add_argument('--all', action='store_true')

# 'report' subcommand
report = subparsers.add_parser('report', help='generate report')
report.add_argument('--format', choices=['text','json','html'])

args = parser.parse_args()
if args.command == 'check':
    run_checks(args.all)
elif args.command == 'report':
    gen_report(args.format)

Challenges & Solutions

  • argparse subparser dest must be set to detect which subcommand was used
  • type=int/float converts argument string — raises argparse error on invalid input

Key Takeaways

  • Always provide help= for every argument — argparse -h depends on it
  • Use choices=[] to restrict valid argument values
