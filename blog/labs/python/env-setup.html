Python Environment Setup

Beginner Python
python3pipvenvpyenvVS Code

Objective

Install Python, create virtual environments, manage packages with pip, and configure a development environment.

Tools & Technologies

  • python3
  • pip
  • venv
  • pyenv
  • VS Code

Key Commands

python3 --version
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip freeze > requirements.txt

Architecture Overview

Step-by-Step Process

01
Install Python

Verify Python 3 is installed and understand pip.

python3 --version
which python3

# Install pip if missing
sudo apt install python3-pip

# Upgrade pip
python3 -m pip install --upgrade pip
02
Create and Use Virtual Environments

venv isolates project dependencies.

# Create
cd myproject
python3 -m venv .venv

# Activate (Linux/Mac)
source .venv/bin/activate

# Activate (Windows)
.venv\Scripts\activate

# Verify — should show .venv path
which python

# Deactivate
deactivate
03
Manage Packages

Install, list, and freeze dependencies.

pip install requests pandas
pip install flask==3.0.0      # specific version
pip list                        # installed packages
pip show requests               # package info
pip freeze > requirements.txt

# Install from requirements
pip install -r requirements.txt

Challenges & Solutions

  • Installing packages globally (without venv) causes version conflicts across projects
  • requirements.txt should be committed to Git — .venv/ should not (add to .gitignore)

Key Takeaways

  • Use one venv per project — never share across projects
  • pyenv lets you manage multiple Python versions side by side
