Step-by-Step Process

01
Numeric Types

Work with integers, floats, and operations.

x = 42          # int
y = 3.14        # float
z = 3 + 4j      # complex

print(type(x))  # 
print(type(y))  # 

# Integer division vs float division
10 // 3   # 3 (floor division)
10 / 3    # 3.333... (float division)
10 % 3    # 1 (modulo)
2 ** 10   # 1024 (power)
02
Strings

Strings are immutable sequences of characters.

name = 'Alice'
greeting = "Hello, World!"
multi = '''Line 1
Line 2'''

# f-strings (Python 3.6+)
age = 25
print(f'{name} is {age} years old')

# Common methods
'hello'.upper()         # 'HELLO'
'  hello  '.strip()     # 'hello'
'a,b,c'.split(',')      # ['a', 'b', 'c']
','.join(['a','b','c']) # 'a,b,c'
03
Type Conversion

Convert between types explicitly.

int('42')       # 42
float('3.14')   # 3.14
str(42)         # '42'
bool(0)         # False
bool('')        # False
bool([])        # False
bool(1)         # True
list('abc')     # ['a', 'b', 'c']

# isinstance for type checking
isinstance(42, int)      # True
isinstance('hi', str)    # True
isinstance(42, (int, float))  # True — check multiple

Challenges & Solutions

  • int('3.14') raises ValueError — must do int(float('3.14'))
  • == compares values; is compares identity — don't use 'is' for value comparison

Key Takeaways

  • Python uses duck typing — what matters is behaviour, not declared type
  • Immutable types (int, str, tuple) are hashable and can be dict keys
