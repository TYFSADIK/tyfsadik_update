Private AI Model

Self-hosted LLM with integrated web search — ai.tyfsadik.org

2026 Infrastructure Live Demo ↗

Overview

This project is a fully self-hosted large language model stack running at ai.tyfsadik.org. At the core is Ollama serving Mistral 7B at the Q4_K_M quantization level, chosen for the optimal balance between inference speed and output quality on CPU-only hardware. The frontend is Open WebUI, a full-featured chat interface that supports conversation history, model switching, and system prompt configuration. SearXNG is integrated as a web search backend, allowing the model to retrieve and cite live information without routing any queries through external cloud APIs.

The entire stack is containerized with Docker Compose and sits behind an Nginx reverse proxy with Let's Encrypt SSL. No data ever leaves the local network for inference — the only outbound traffic is SearXNG fetching search engine results, which it does without exposing any user identity. This setup eliminates OpenAI subscription costs, preserves full data privacy, and keeps model weights under local control.

Tech Stack

  • Ollama — local LLM runtime, handles model loading and inference API
  • Mistral 7B (Q4_K_M) — 4-bit quantized model, ~4.1 GB VRAM/RAM footprint
  • Open WebUI — full-featured chat UI with history, model switcher, system prompts
  • SearXNG — self-hosted meta-search engine, provides web context to the LLM
  • Docker & Docker Compose — container runtime and orchestration
  • Nginx — reverse proxy, CORS headers, SSL termination
  • Let's Encrypt / Certbot — automated TLS certificate provisioning and renewal
  • Linux (Debian 12) — host operating system

Build Process

1

Install Ollama and Pull Model

Ollama is installed via its official install script, which sets up the binary and a systemd service. After installation, the Mistral 7B model at Q4_K_M quantization is pulled from the Ollama model registry. This quantization level reduces the model to approximately 4.1 GB while keeping perplexity within acceptable bounds for general use. 

# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Verify service is running
systemctl status ollama

# Pull Mistral 7B at Q4_K_M quantization
ollama pull mistral:7b-instruct-q4_K_M

# Test inference locally
ollama run mistral:7b-instruct-q4_K_M "Hello, world"
2

Deploy Open WebUI

Open WebUI is deployed as a Docker container. The --add-host flag is required so that the container can reach Ollama running on the host (not inside Docker). A named volume persists conversation history and user settings across container restarts. 

docker run -d \
  -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
3

Deploy SearXNG via Docker Compose

SearXNG is cloned from its official Docker configuration repository. The settings.yml file is edited to generate a unique secret key, disable search tracking, and enable preferred search engines. The service is started on port 8080. 

git clone https://github.com/searxng/searxng-docker.git
cd searxng-docker

# Generate secret key and configure
sed -i "s|ultrasecretkey|$(openssl rand -hex 32)|g" searxng/settings.yml

# Disable rate limit for local use
# Edit searxng/settings.yml: server.limiter: false

docker compose up -d

# Verify SearXNG is responding
curl http://localhost:8080/search?q=test&format=json
4

Configure Nginx Reverse Proxy

Nginx is configured to proxy requests for ai.tyfsadik.org to the Open WebUI container. CORS headers are added to allow the WebUI JavaScript to call the SearXNG API from the browser context. WebSocket support is enabled for real-time streaming responses. 

server {
    listen 80;
    server_name ai.tyfsadik.org;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;

        # CORS for SearXNG integration
        add_header Access-Control-Allow-Origin "https://ai.tyfsadik.org" always;
        add_header Access-Control-Allow-Methods "GET, POST, OPTIONS" always;
        add_header Access-Control-Allow-Headers "Content-Type, Authorization" always;
    }
}
5

Provision SSL Certificate

Certbot provisions a Let's Encrypt certificate and automatically modifies the Nginx config to redirect HTTP to HTTPS and serve on port 443. A cron job (installed by certbot) handles automatic renewal every 60 days. 

apt install certbot python3-certbot-nginx -y

certbot --nginx -d ai.tyfsadik.org

# Verify auto-renewal timer
systemctl status certbot.timer

# Test renewal dry run
certbot renew --dry-run
6

Connect Open WebUI to SearXNG

In the Open WebUI admin panel, navigate to Settings → Web Search and set the SearXNG URL to the local container address. This enables the model to perform web searches inline during conversations. The integration uses SearXNG's JSON API endpoint. 

# SearXNG JSON API endpoint for Open WebUI:
# http://localhost:8080/search?q={query}&format=json

# Test the integration endpoint
curl "http://localhost:8080/search?q=current+date&format=json" | jq '.results[0]'

# In Open WebUI: Settings > Web Search
# Web Search Engine: SearXNG
# SearXNG Query URL: http://localhost:8080/search?q=<query>&format=json

Challenges & Solutions

  • CPU inference latency: Running Mistral 7B at full precision on a CPU produced 3–5 tokens/sec, making conversations frustrating. Switching to Q4_K_M quantization cut model size from ~14 GB to ~4.1 GB and raised throughput to 12–18 tokens/sec with no perceptible drop in response quality for conversational tasks.
  • CORS errors between WebUI and SearXNG: When Open WebUI's JavaScript attempted to hit the SearXNG JSON endpoint directly, browsers blocked the cross-origin request. Resolved by routing all SearXNG traffic through a dedicated /search location block in Nginx, which adds the correct Access-Control-Allow-Origin headers.
  • Ollama not reachable from Docker container: Open WebUI running inside Docker could not reach Ollama on the host by default. Fixed by passing --add-host=host.docker.internal:host-gateway at container startup and setting OLLAMA_BASE_URL=http://host.docker.internal:11434.
  • Streaming response timeouts: Long model outputs caused Nginx to drop connections before the response completed. Resolved by setting proxy_read_timeout 300s and proxy_send_timeout 300s in the Nginx location block.

What I Learned

  • LLM quantization formats (GGUF, Q4_K_M, Q8) and their RAM/quality trade-offs
  • Nginx reverse proxy configuration for WebSocket and SSE streaming
  • Docker host networking and inter-container communication patterns
  • SearXNG engine configuration, rate limiting, and JSON API usage
  • Let's Encrypt certificate management and automated renewal with systemd timers
  • The practical difference between RAG (retrieval-augmented generation) and simple web-search injection
